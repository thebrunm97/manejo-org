# Planning Phase (Step-by-Step Strategy)

Passo 1: Baseado no Context Summary gerado no `/research`, crie um plano de ação estrito.
Passo 2: Quebre o plano em "Fases" (ex: Fase 1: Atualizar Structs, Fase 2: Atualizar FSM).
Passo 3: Para cada fase, liste os arquivos que serão modificados.
Passo 4: Apresente o plano ao usuário em formato de Markdown com Checkboxes `[ ]`.
Passo 5: Pergunte: "Você aprova este plano? Se sim, digite /implement para começarmos a Fase 1."
